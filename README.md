# 项目要求

## 一、 场景与功能描述

### 1. 游戏场地
* 设计一个较大的棋盘作为游戏场地。
* 场地中包含 **甲方** 和 **乙方** 各自的一个基地。
* 棋盘中分布有 **山川、河流** 等障碍物。
* 任何运动的人（单位）不能从障碍物穿过。

### 2. 基地机制
* 基地能够 **自动造出** 士兵。
* 需要设计 **三种不同类型** 的士兵。
* 基地拥有较大的 **血量**。

### 3. 士兵性质
* **基础指标**：攻击力、攻击范围、移动速度、血量、盔甲厚度、视野范围等。
* **自主决策**：士兵可以依据 **视野范围内** 的场景独立作出决定，进行移动或攻击等操作。

---

## 二、 基本开发要求

### 1. 独立完成
* 请独立完成上述系统。
* 需要决策的地方，可以采用 **随机方法** 决定作为基础解。

### 2. 输出方式
* 至少以 **字符方式（Text-based/Console）** 输出目前的状态。

### 3. 胜利条件
* 最终攻破对方基地的一方为胜利。
* 请输出胜利方法/结果。

---

## 三、 扩充要求（酌情添加）

为了获得更好的评价或挑战自我，可尝试以下扩展功能：

* **图形化显示**：利用图形化显示技术，图形化显示当前实时状态（GUI）。
* **多线程技术**：利用多线程技术，实现对象的线程化。
* **智能决策**：加入决策能力（AI算法），替代基础的随机决策。
* **人工干预**：可加入人为的操纵（如利用键盘输入控制，或图形化界面控制等）。
